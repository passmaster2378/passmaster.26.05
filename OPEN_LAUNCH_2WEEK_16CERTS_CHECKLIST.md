# PASSmaster · 15과정 완전 오픈 목표 · 2주 집중 체크리스트

**목표:** 메인 노출 **15종 자격 과정**(슬러그 `forklift`·`excavator`·`electric`·`welding`·`hazmat`·`carrepair`·`beautician`·`makeup`·`skin`·`nail`·`elevator`·`cookkr`·`cookwest`·`cookcn`·`cookjp`)에 대해 **수강·결제 흐름·정보 페이지·네비 연결이 끊기지 않도록** 한다. 학습 앱은 메이크업만 구현되어 있으므로, “완전 오픈”의 정의에는 **판매 가능 + 연결 무결성**을 우선 포함하고, **문제은행 학습 UX**는 과정별 상태를 명시적으로 구분한다.

**참조 문서:** `OPEN_LAUNCH_CHECKLIST.md`(운영/스모크), `backend/server.js`의 `CANONICAL_PASSMASTER_COURSES`, `assets/auth-client.js`의 `CERT_SLUG_LABEL_MAP`, `cert/info.html`의 `CERT_GENERAL`.

## 개선 원칙 (2주 안에 처리할 우선순위)

1. **단일 출처(SSOT) 명시**: 슬류그 목록은 `CERT_SLUG_LABEL_MAP` 또는 파생 목록 하나로 두고 FE/카피/`cert` 페이지·백업 문서 동기화.
2. **가시성 높은 URL 계약 고정**: `enroll/index.html?cert=<slug>#enroll-openings`, `cert/info.html?slug=<slug>` 는 카드·푸터·내비에서 같은 규칙으로 재사용.
3. **“학습 제공 범위”를 UI 텍스트로 고지**: 과정별로 학습 제공/준비중/메이크업만 체험 등을 카드 또는 공통 패턴으로 처리해 기대 불일치 감축.
4. **메이크업 외 14종**: 최소 준비 **수강+정보+Q-Net** 연결 검증부터 막고, 문제은행은 **예정 페이지** 또는 **공통 허브**로 우선 차단 불만 제거.

## 마스터 체크리스트 (50항목 이상)

### A. 과정 목록 및 데이터 정합성 (15종 동기화)

- [ ] **A1.** `CERT_SLUG_LABEL_MAP` 에 정의된 15개 키가 `CANONICAL_PASSMASTER_COURSES[].code` 와 문자열 포함 대소문자까지 일치함.
- [ ] **A2.** 운영 DB에서 `courses` 테이블에 위 15 `code` 가 모두 존재하고 `status` 가 의도와 일치(오픈 시 `open`).
- [ ] **A3.** 각 과정에 대해 **`application_status` 가 `open` 또는 `closing` 인 `course_openings`** 가 존재하며, 과거 레거미 IS/EE/IT 만 남아 신규 과정 노출을 가리지 않음.
- [ ] **A4.** `GET /api/course-openings` 결과에 **15 과정 각각 최소 1건** 해당하는 모집이 포함되는지 검증(API 직호출 또는 브라우저 네트워크 탭).
- [ ] **A5.** `openingMatchesCertSlug` 규칙(`code===slug` 또는 `title.includes(한글라벨)` )으로 **각 슬류그 선택 시 해당 과정 행만** 남거나, 과정별로 명확히 구분 가능함.
